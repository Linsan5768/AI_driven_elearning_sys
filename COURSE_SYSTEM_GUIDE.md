# 🎓 智能学习系统使用指南

## 🌟 系统概述

这是一个基于LLM的智能学习系统，将游戏化元素与计算机科学教育内容相结合。每个区域代表一个学习主题，用户必须通过知识测试才能解锁下一个区域。

## 📚 课程内容配置

## 🔧 自定义课程内容

### 添加新课程

在 `frontend/src/config/courseMaterials.ts` 中添加新的课程：

```typescript
'area6': {
  subject: '人工智能基础',
  materials: [
    '机器学习：监督学习、无监督学习、强化学习',
    '神经网络：输入层、隐藏层、输出层',
    '深度学习：卷积神经网络、循环神经网络',

  ],
  difficulty: 'hard',

```

## 🎯 学习流程

### 第一阶段：对话交流
1. 进入区域后，AI导师会欢迎你
2. 你可以与AI导师进行课程相关对话
3. 询问概念、请求解释、讨论应用场景

### 第二阶段：知识测试
1. 对话进行到一定程度后，系统自动生成测试题目
2. 题目基于该区域的课程资料
3. 提供4个选项，选择正确答案

### 第三阶段：结果反馈
1. 提交答案后立即显示结果
2. 正确答案会高亮显示
3. 提供详细的解释说明
4. 答对可以继续，答错需要重新学习

### 第四阶段：解锁区域
1. 只有答对题目才能完成任务
2. 系统标记该区域为已完成
3. 自动生成通往新区域的路径

## 🧠 LLM智能出题

### 出题逻辑
1. **基于课程资料**：题目内容完全来自预设的课程材料
2. **智能生成**：LLM根据材料内容生成相关题目
3. **格式验证**：确保题目格式正确，包含所有必要信息
4. **备用方案**：如果LLM生成失败，使用预设题目

### 题目类型
- **概念理解**：考察对基本概念的理解
- **应用分析**：考察知识点的实际应用
- **比较辨析**：考察不同概念的区别
- **综合运用**：考察多个知识点的综合运用

### 质量保证
- **数学题验证**：防止题目泄露答案、多个正确答案、选项格式错误
- **逻辑一致性**：确保解析与正确答案一致
- **选项唯一性**：验证只有一个正确答案
- **重试机制**：LLM生成失败时自动重试

## 💡 使用建议

### 学习策略
1. **循序渐进**：按区域顺序学习，打好基础
2. **主动提问**：与AI导师多交流，深入理解概念
3. **实践应用**：思考知识点的实际应用场景
4. **复习巩固**：答错题目后重新学习相关材料

### 技术配置
1. **本地优先**：推荐使用Ollama本地模型
2. **网络备用**：Hugging Face作为云端备选
3. **定期更新**：保持模型和课程内容的最新状态
